# Generate PRD Skill
## Rules
- When asked to generate a PRD, always use the template below.
- Keep descriptions concise but comprehensive.
- Ensure functional and non-functional requirements are clearly separated.

## PRD Template
### 1. Project Description
- **Problems:** Main issues or challenges the project aims to address.
- **Aims & Objectives:** Specific outcomes, improvements, or advancements.

### 2. Scope of the Project
- **Project Deliverables:** What exactly is being delivered.
- **Project Exclusions:** What the project doesn’t include.
- **Project Constraints:** Time, scope, risk, and resources.
- **Assumptions:** Core assumptions made before development.

### 3. Requirements Specification
- **Functional Requirements:** What the system must do.
- **Non-Functional Requirements:** Performance, security, and usability metrics.
